# PRD: Skill Generation

## Purpose

`docs/ai/AI_AGENTS_SKILLS_MCP_STRATEGY.md` says Recall OS should generate AI agent skills, but the
tool does not yet produce them. This feature makes Recall OS generate the documented skill set so a
repository actually gets the durable workflows its AI agents should follow — and so Recall OS can
generate its own skills instead of hand-authoring them.

## Problem

- Skills are described in strategy memory but never generated by the tool.
- A repository adopting Recall OS gets no reusable agent workflows.
- Recall OS's own skills are hand-authored, which contradicts the dogfooding thesis.

## In Scope

- Add a `recall skill create <name>` command that generates a valid Agent Skills `SKILL.md`.
- Emit to both `.claude/skills/<name>/SKILL.md` and the portable `.agents/skills/<name>/SKILL.md`.
- Ship a built-in catalog of the documented MVP skills with research-validated format and "Use when"
  trigger descriptions.
- Generate a valid skeleton for unknown skill names.
- Add a `recall skill list` command.
- Add tests and docs.

## Non-Goals

- No scripts inside generated skills (MVP rule).
- No Claude Code-only fields or dynamic shell injection (keeps skills portable).
- No runtime agent execution.

## Users

- Repositories adopting Recall OS that want reusable AI workflows.
- AI agents that should follow the same disciplined workflows the project uses.
- Maintainers dogfooding the skill set.

## Success Criteria

- `recall skill create write-tests` writes a valid `SKILL.md` to both targets.
- Generated skills validate against the Agent Skills format (name regex, description length, "Use
  when" trigger).
- Generated skills contain no scripts and no fenced code blocks.
- Unknown names produce a valid skeleton.
- Recall OS's own skills can be generated from the catalog.
