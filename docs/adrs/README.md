# Architecture Decision Records

ADRs record accepted or proposed architecture decisions.

Use ADRs when changing:

- Architecture style.
- File write policy.
- Security posture.
- Dependency policy.
- Runtime network behavior.
- Telemetry behavior.
- MCP from guidance-only to active workflow dependency.
- Plugin or preset execution model.

Use `ADR_TEMPLATE.md` for new ADRs.
